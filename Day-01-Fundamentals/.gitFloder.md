## The .git Folder – Git's Brain

# Structure of .git/
.git/
├── objects/              ← Stores all commits, blobs, trees
│   ├── ab/
│   ├── cd/
│   ├── ef/
│   ├── pack/
│   └── info/
├── refs/                 ← References to branches and tags
│   ├── heads/
│   │   ├── main          ← Points to main branch commit
│   │   └── feature       ← Points to feature branch commit
│   ├── remotes/
│   │   └── origin/
│   │       └── main
│   └── tags/
├── HEAD                  ← Points to current branch (main)
├── config                ← Repository settings (user, remote URL)
├── description           ← Repository description
├── hooks/                ← Scripts that run on Git events
├── index                 ← Staging area (files ready to commit)
└── logs/
    └── HEAD              ← Log of all HEAD movements

## What Each Part Does
| File/Folder |	Purpose                                          |
|-------------|--------------------------------------------------|
| objects/    |	Stores actual commit data as compressed files    |
| refs/	      | Points to branches (e.g., main, feature)         |
| HEAD	      | Shows which branch you're currently on           |
| config	  | Your Git settings (username, email, remote URL)  |
| index	      | Staging area - files waiting to be committed     |
| hooks/	  | Scripts for automation (pre-commit, post-commit) |

## CRITICAL: Do NOT Modify .git Folder!
