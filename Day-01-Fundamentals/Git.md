## Git = A distributed **version control system** that tracks changes in your code over time.

## Why Do You Need Git?

| Problem                | Without Git                                  | With Git                        |
|------------------------|:--------------------------------------------:|:-------------------------------:|
| Lost Code              | File gets deleted → Lost forever             | Can recover from history        |
| Multiple Versions      | Have file_v1, file_v2, file_final... chaos   | One file, infinite versions     |
| Team Collaboration     | Email files back and forth → Merge hell      | Merge changes intelligently     |
| Who Changed What?      | No idea who modified which line              | See exact commit + author       |
| When Did Bug Happen?   | Unknown                                      | Find exact commit that broke it |

## Git vs GitHub

GIT                          GITHUB
┌──────────────────┐        ┌──────────────────┐
│ Version Control  │        │ Cloud Platform   │
│ Tool (local)     │   →    │ Hosting Service  │
│ Tracks changes   │        │ Web interface    │
│ Command line     │        │ Collaboration    │
│ Free & Open      │        │ Public/Private   │
│ Works offline    │        │ Requires internet│
└──────────────────┘        └──────────────────┘
